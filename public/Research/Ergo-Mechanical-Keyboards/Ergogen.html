    Ergogen

    Jun 13, 20244 min read

    • programming
    • keyboards

    Ergogen is a tool for systematically and parametically creating keyboard designs. I’ve used it in the past, but it has changed a lot so I’m basically relearning the tool from https://flatfootfox.com/ergogen-part1-units-points/.

    Part 1

    Units

    Units/variables are critical for any good program! One of the YAML sections in Ergogen is for units. Ergogen has 4 basic units:

    • u = 19.00mm = MX switch spacing
    • U = 19.05 mm = MX keycap spacing
    • cx = 18mm = choc x spacing
    • cy = 17mm = choc y spacing.

    Fox strongly suggests using these “proxy” variables.

    Points

    Points describe where each key is. A single Point has the [x, y] that represents the coordinate at the center of the a key, and a r value describing the rotation. A group of points is a Zone. Here is an example 3x5 matrix:

    You might want to skip certain keys, which you can do like:

    columns:
	pinky:
		rows.bot.skip: true

    Everything we’ve created so far is just a rectangle though. Use key.stagger to adjust the stagger of a column, relative to the previous one. You can use key.splay to adjust columnar rotation relative to the previous column. You can also adjust settings for a whole zone using the zones.MyZone.key field:

    It is typical to define a second zone of keys for a thumbcluster. You can use the zones.MyZone.anchor field to tell Ergogen the starting point for a zone.

    ref: is a reference of where the anchor should be placed. The matrix_inner_bot follows the syntax of zone_column_row. You can also shift from the anchor ref, shown with the shift: key.

Anywhere that rectangles overlap or touch edges get combined into one shape. With an outline like this though the thumb keys could easily get left out. Just double check it.
